Lil Agents 改造方案

安全优化 + Hilda 西高地小狗 + 独立 Agent 绑定

A

轻量改造

安全：在现有菜单加一个 "Safe Mode / Full Auto" 开关，Safe Mode 下去掉危险 flag

Hilda：直接在 LilAgentsController 里加第三个 WalkerCharacter，复用现有架构

Agent 绑定：给每个角色加一个右键菜单，可以选择该角色使用哪个 AI 后端

改动量：~6 个文件，最小侵入

优点

  • 改动最少，风险低
  • 保持原有代码风格

缺点

  • 菜单会变臃肿（3 角色 × 各自设置）
  • 角色配置散落在多处
B

结构化改造（推荐）

安全：新增 PermissionMode enum（restricted / fullAuto），每个 Session 类根据模式决定 flag

Hilda：引入 CharacterConfig 结构体，把角色定义（视频、颜色、偏移、走路参数、agent、人格）数据化

Agent 绑定：每个 CharacterConfig 持有自己的 AgentProvider，菜单按角色分子菜单

改动量：~8 个文件，中等重构

优点

  • 加第 4、5 个角色零额外成本
  • 角色配置集中管理
  • 菜单结构清晰

缺点

  • 需要重构 Controller 和 AppDelegate
C

完整重构

安全：加沙箱、环境变量过滤、权限确认 UI

Hilda：做一个角色编辑器，支持用户自己导入视频和配置

Agent 绑定：完全插件化的 agent 系统

改动量：大量新文件，大重构

优点

  • 最大灵活性

缺点

  • 工程量大，偏离"好玩"的初衷
  • 过度设计
